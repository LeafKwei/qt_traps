# 在`.cpp`文件中插入Q_OBJECT宏导致编译失败

有时我们在测试时为了图方便，可能会将测试类及其函数直接定义在源文件中，在C++中这可能没太大问题，但是在QT里，由于我们可能需要用到QT的元对象系统，所以需要在类定义里插入`Q_OBJECT`宏，例如：

```c++
/* main.cpp */
class Test : public QObject{
	Q_OBJECT
public:
	void test(){
		/* some works */
	}
};

int main(int argc, char *argv[]){
	Test().test();
}
```

如果是原生的C++，那么这段程序并没有太大的问题，但是在QT里面，由于MOC工具不会扫描`.cpp`文件，所以会使得`.cpp`中插入的`Q_OBJECT`被忽略，自然也就不会为其生成经过MOC处理后的文件，从而导致编译、链接阶段发生错误。出现该错误时，QT的错误提示只有一条玄学的`subcommand failed`，让人无从着手。

知道了原因，解决这个问题就很简单了，只需要把类定义放到`.h`或`.hpp`结尾的头文件中即可：

```c++
/* Test.hpp */
class Test: public QObject{
	Q_OBJECT
public:
	void test(){
		/* some works */
	}
};
```
